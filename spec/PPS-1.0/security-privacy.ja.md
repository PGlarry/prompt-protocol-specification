# 1. 脅威モデル
- 注入（プロンプト / URL / エビデンス汚染）
- 越権（未宣言のツールまたは外部チャネル）
- コンプライアンスリスク（PII、引用漏れ、管轄域をまたぐ制限）

# 2. 対策（規範的参照）
- インターフェース白名単 & 能力サンドボックス：`how_to_do.tools ⊆ who.capabilities`（REQ-181）
- 外部ブラウズ禁止制約：`no_external_browse` ⇒ `web_browse` を無効化し URL を除去（REQ-052/183）
- 引用一貫性：`citations_required` ⇒ `evidence` 非空（REQ-180）
- GDPR：`gdpr` ⇒ `no_pii`（REQ-182）
- 因果的非干渉：未認可入力が認可された出力の分布を変えてはならない（論文本文および規範仕様に対応）

# 3. 残余リスク & 境界
- サードパーティモデルや検索ソースの制御不能な更新による軽微なドリフト（決定論的デコードとエビデンススナップショットで緩和）
- 人手アノテーションや外部スコアリングの主観性（KPI 閾値と Schema 検証で軽減）
