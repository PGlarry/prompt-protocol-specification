# 1. 위협 모델
- 주입 (프롬프트 / URL / 증거 오염)
- 초과 권한 (미선언 도구 또는 외부 채널)
- 준수 위험 (PII, 누락된 인용, 관할권 간 제한)

# 2. 대응 방안 (규범적 참조)
- 인터페이스 허용 목록 & 역량 샌드박스: `how_to_do.tools ⊆ who.capabilities` (REQ-181)
- 외부 브라우징 금지 제약: `no_external_browse` ⇒ `web_browse` 비활성화 및 URL 제거 (REQ-052/183)
- 인용 일관성: `citations_required` ⇒ `evidence` 비어있지 않음 (REQ-180)
- GDPR: `gdpr` ⇒ `no_pii` (REQ-182)
- 인과적 비간섭: 미승인 입력이 승인된 출력의 분포를 변경해서는 안 됨 (논문 본문 및 규범 사양에 대응)

# 3. 잔여 위험 & 경계
- 타사 모델 또는 검색 소스의 통제 불가능한 업데이트로 인한 경미한 드리프트 (결정론적 디코딩 및 증거 스냅샷으로 완화)
- 수동 주석 또는 외부 점수화의 주관성 (KPI 임계값 및 스키마 검증으로 경감)
