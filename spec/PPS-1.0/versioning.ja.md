# 1. セマンティックバージョニング
- `MAJOR.MINOR.PATCH` を採用する。プロトコルバージョンは `header.pps_version` に `PPS-vMAJOR.MINOR.PATCH` として記載する。

# 2. 互換性ポリシー
- 1.0.x シリーズ：新規フィールドはオプションを維持し、後方互換性を保つ。
- MAJOR の引き上げは破壊的変更（フィールド削除または意味変更）の場合のみ行う。

# 3. 移行
- 1.0 から 1.x へのアップグレード：サンプルと CI に対象バージョンを明記する。新しいポリシー条項が追加された場合は、`policy_checks` のルールアップグレードで検出できる。
