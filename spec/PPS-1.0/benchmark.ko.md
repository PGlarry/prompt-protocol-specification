# 1. 목적
프로토콜 계층의 역량을 정량화합니다: 재현성, 거버넌스 효과성, 리소스 명시성.

# 2. 메트릭
- 정책 포착률 = FAIL 수 / 총 케이스 수 (유형별: 주입, 초과 권한, 인용, GDPR, 예산)
- 자동 수정 성공률 = 수정 후 PASS 수 / 자동 수정이 트리거된 케이스 수
- 재생 일관성률 = 다중 플랫폼 재생에서 일관된 (또는 허용 범위 내) 비율
- 리소스 예산 명시율 = `how_much` 예산 필드를 포함하는 케이스 비율

## 2.1 부분 사양 / 자유 조합 절제 실험
- 입력 설정: `what.task`만 제공; 나머지 7개 차원은 무작위로 하위 집합을 선택하여 시스템이 완성 (REQ-341..343 준수).
- 평가 항목:
  - 준수 유지율: 완성 후에도 교차 필드 불변 조건 (REQ-180..183)을 만족하는 비율
  - 정렬 견고성: 제공된 차원 수 (0..7)에 따른 KPI 통과율 곡선
  - 예산 편차: 완성된 리소스 예산과 목표 구성 간의 편차 분포
