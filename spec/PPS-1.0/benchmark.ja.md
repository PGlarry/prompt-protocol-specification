# 1. 目的
プロトコル層の能力を定量化する：再現性、ガバナンス有効性、リソースの明示性。

# 2. メトリクス
- ポリシー捕捉率 = FAIL 数 / 総ケース数（タイプ別：注入・越権・引用・GDPR・予算）
- 自動修復成功率 = 修復後 PASS 数 / 自動修復が発動したケース数
- リプレイ一致率 = マルチプラットフォームリプレイで一致（または許容差内）した割合
- リソース予算明示率 = `how_much` 予算フィールドを含むケースの割合

## 2.1 部分仕様／自由組み合わせアブレーション
- 入力設定：`what.task` のみを提供；残り 7 次元についてはランダムにサブセットを選択しシステムに補完させる（REQ-341..343 に従う）。
- 評価項目：
  - コンプライアンス維持率：補完後もクロスフィールド不変条件（REQ-180..183）を満たす割合
  - 整合堅牢度：提供次元数（0..7）に対する KPI 達成率の曲線
  - 予算偏差：補完で生成されたリソース予算と目標設定との偏差分布

# 3. データソース
- `tests/pps-conformance/policy_checks.js --json` の `issues[].type`
- `auto_fix.js` の `fixesApplied`（後続スクリプトがエクスポート）
- `summary.js` が集約して `summary.csv` と `benchmark.json` を生成
